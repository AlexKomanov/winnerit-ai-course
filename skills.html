מה זה Skill?

Skill הוא קובץ Markdown עם הוראות מובנות שמנחות את Claude Code למשימות ספציפיות — בדיקות, debugging, code review, ועוד.

במקום לכתוב בכל פעם את אותן ההנחיות מחדש, מגדירים skill אחד ומפעילים אותו עם /skill-name.

reusable · composable · team-shareable

מרכיבי הSkill

1 name

שם

Slug

שם קצר ב-kebab-case — זה מה שמקלידים ב-/skill-name. מזהה ייחודי בתוך הפרויקט.

2 trigger

טריגר

When to Use

מתי להפעיל את ה-Skill — תנאים, קונטקסט, או סיטואציות שבהן הוא רלוונטי.

3 description

תיאור

Description

משפט אחד שמסביר מה ה-Skill עושה — עוזר ל-AI להחליט מתי להפעיל אוטומטית.

4 body

גוף

Instructions

ההוראות עצמן — תהליך, checklist, דוגמאות. ה-AI עוקב אחריהן צעד אחר צעד.

הגדר.
הפעל.

Skill שנכתב פעם אחת יכול לשפר כל session — לבד, לצוות, לכל פרויקט.

משתמשים בSkill

1 find

מגלים skills זמינים

בכל session ניתן לראות אילו skills מוגדרים בפרויקט — ב-CLAUDE.md או בתיקיית .claude/.

→ /help — רשימת הפקודות הזמינות
2 invoke

מפעילים את ה-Skill

מקלידים / ואז שם ה-Skill. Claude Code טוען את ההוראות ומתחיל לפעול לפיהן.

→ /code-review — מפעיל skill לבדיקת קוד
3 follow

עוקבים אחרי התהליך

ה-Skill מנחה את Claude Code — הוא עשוי לשאול שאלות, להריץ פקודות, ולספק checklist ברור.

→ Claude Code עוקב אחרי ה-checklist

יוצרים Skill חדש

1 create

יוצרים קובץ

יוצרים קובץ .md בתיקיית .claude/ בפרויקט. שם הקובץ הוא שם ה-Skill.

>_ mkdir -p .claude && touch .claude/my-skill.md
2 define

כותבים את ה-Skill

מוסיפים frontmatter עם name, description, ו-when_to_use — ואז את גוף ההוראות ב-Markdown.

→ frontmatter + הוראות מפורטות ב-Markdown
3 test

מנסים ומשפרים

מפעילים את ה-Skill ב-Claude Code ובודקים שהוא עושה מה שרצינו. מעדכנים את הקובץ לפי הצורך.

→ /my-skill — מריצים ובודקים
