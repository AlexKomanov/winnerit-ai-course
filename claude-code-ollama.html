Claude Code מדבר עם Ollama.

Claude Code שולח בקשות ל-API — בדרך כלל לשרתי Anthropic בענן. כשנגדיר שלושה משתני סביבה פשוטים, הוא ישלח את אותן הבקשות ל-Ollama המקומי שרץ במחשב שלנו. שום דבר לא עוזב את המכשיר.

claude code → ollama api → local model

למה מקומי?

פרטיות מוחלטת

הקוד, הפרויקטים, השיחות — הכל נשאר במחשב שלנו. שום מידע לא נשלח לשרתים חיצוניים.

ללא עלויות

אין tokens, אין חשבונית חודשית, אין rate limits. לאחר הורדת המודל — שימוש חינמי לנצח.

עצמאות מלאה

עובד offline, ללא תלות בשרתי Anthropic, ללא הגבלות. אנחנו שולטים בכלי שלנו.

חומרה.
זיכרון.
ביצועים.

המחשב הוא המוח של המודל. ה-RAM שלנו קובע אילו מודלים נוכל להריץ — ואת איכות התשובות שנקבל.

context window · minimum requirement

Claude Code דורש לפחות
בחלון ההקשר
64,000 tokens.

Claude Code שולח הקשר ארוך מאוד — קוד, קבצים, היסטוריית שיחה ומידע על הפרויקט. מודל עם חלון קטן מ-64k יכשל. כל המודלים שהצגנו עומדים בדרישה הזאת.

Settings → Context Length → 64k

המלצה לרוב המשתמשים: Mac עם 16GB unified memory מריץ את qwen3.5:9b בנוחות. לביצועים גבוהים יותר — gpt-oss:20b על 24GB. ל-glm-5.1 דרושים לפחות 32GB.

חיבור בפקודה אחת

ollama launch claude מגדירה את כל משתני הסביבה ומפעילה את Claude Code — פקודה אחת, ללא שום הגדרה נוספת. אם המודל לא קיים עדיין, הוא יורד אוטומטית.

מצב ענן — מודלים ענקיים

חלק מהמודלים גדולים מדי בשביל לרוץ מקומית. ל-Ollama יש שירות ענן שמריץ אותם בשבילנו — דרך אותה ממשק בדיוק, רק עם תגית :cloud.

120B+ פרמטרים

גישה למודלים שלא נכנסים בזיכרון המקומי — ביצועים מקסימליים ללא השקעה בחומרה.

אותה ממשק
