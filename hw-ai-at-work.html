התקנת Plugins

1 how to install

איך מתקינים plugin

נפתח את לוח ה-Plugins, נחפש ונתקין ב-user scope.

/plugin → Discover → Search and choose → Install for you (user scope)
2 plugins

Plugins להתקין

· superpowers · code-review · frontend-design · skill-creator
3 reload

טוענים מחדש

לאחר ההתקנה, נריץ reload כדי שכל ה-Plugins יהיו פעילים.

› /reload-plugins

Documentation Skill

1 skill-creator

יוצרים Documentation Skill

נשתמש ב-skill-creator ליצירת ה-Skill. הוא יופעל אוטומטית על בסיס ההקשר — נתאר מה ה-Skill צריך לעשות:

› Create a documentation skill that reads the project structure and key files, then generates clear markdown documentation describing the architecture, main components, and how to run the project
2 scope

שומרים ב-user scope

skill-creator ישאל איפה לשמור את ה-Skill — נבחר user scope כדי שיהיה זמין בכל הפרויקטים על המחשב (~/.claude/skills/).

3 use

מפעילים את ה-Skill

לאחר היצירה, נוכל להריץ אותו בכל פרויקט:

› Generate documentation for this project

פיצ׳ר חדש עם Agent

1 create

יוצרים Feature Writer Agent

/agents → Library → Create new agent
2 capabilities

מגדירים את יכולות של הסוכן שניצור

  • · מושך קוד עדכני מ-GitHub
  • · שואל על הפיצ׳ר הרצוי
  • · סורק את מבנה הפרויקט
  • · frontend-design + superpowers לתכנון
  • · code-review על הקוד
  • · מציג שינויים לבדיקה
  • · commit + push לאחר אישור
3 settings

מגדירים את ה-Agent

  • · User scope — זמין בכל הפרויקטים
  • · Color — צבע ויזואלי לזיהוי
  • · Confirmation — מאשרים לפני שה-Agent נוצר
4 invoke

מפעילים את ה-Agent

מתארים את הפיצ׳ר — ה-Agent יעבור את כל השלבים.

› Use the feature writer agent to add [describe your feature] to the app
5 approve

מאשרים את השינויים

ה-Agent יציג שינויים לביקורת — לאחר אישורנו יבצע commit + push אוטומטית.

Playwright MCP + PRD

1 validate

מוודאים חיבור MCP

נוודא ש-Playwright MCP מחובר ופעיל ב-Claude Desktop לפני שמתחילים.

Claude Desktop → Settings → Developer → Local MCP servers → Playwright ✓
2 explore + prd

סיור ויצירת PRD

בבקשה אחת נבקש מ-Claude Desktop לסייר באפליקציה עם Playwright MCP — לעבור על כל הדפים, הכפתורים והטפסים — ולאחר מכן לכתוב PRD מלא הכולל סקירת מוצר, זרימות משתמש, רשימת פיצ׳רים ותרחישי בדיקה, ולשמור אותו כקובץ Markdown או Doc.

Gem לבדיקות

1 create gem

יוצרים Gem ב-Gemini

ניצור Gem חדש שיהיה סוכן לייצור test cases. נוסיף את ה-PRD מהשלב הקודם כ-resource — או כל PRD קיים אחר — ונגדיר את תפקידו כסוכן QA.

2 generate

מייצרים Test Cases

נבקש מה-Gem לייצר test cases על בסיס ה-PRD — כבר יש לו את כל ההוראות הדרושות.

› Create several tests for the [feature name] feature
