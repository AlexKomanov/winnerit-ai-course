מה זה Git?

Git הוא מערכת לניהול גרסאות — הוא עוקב אחרי כל שינוי בקוד ומאפשר לנו לחזור לכל נקודה בזמן, לעבוד במקביל ולמזג שינויים בבטחה.

Git עובד עם שלושה אזורים — Working Directory, Staging Area, ו-Repository.

פקודות ליבה

1 clone

clone

מורידים עותק מלא של repository מ-GitHub למחשב — כולל כל ההיסטוריה.

2 add

add

מסמנים אילו קבצים לכלול ב-commit הבא — רק מה שנסמן יישמר.

3 commit

commit

שומרים snapshot של השינויים עם הודעה תיאורית — כל commit הוא נקודת חזרה.

4 push / pull

push / pull

push שולח commits ל-GitHub. pull מביא שינויים חדשים מ-GitHub אלינו.

משכפלים repository

1 copy url

מעתיקים את ה-URL

ב-GitHub נכנסים לדף הפרויקט, לוחצים Code → HTTPS ומעתיקים את הכתובת.

GitHub · Code · HTTPS · Copy URL
2 clone

משכפלים

מריצים את הפקודה בטרמינל — Git יוצר תיקייה עם כל הקוד וההיסטוריה.

>_ git clone https://github.com/user/repo.git
3 enter

נכנסים לתיקייה

Git יצר תיקייה בשם הפרויקט. נכנסים אליה ומוכנים לעבוד.

>_ cd repo-name && ls

שומרים שינויים

1 status

בודקים מצב

רואים אילו קבצים השתנו — קבצים אדומים עדיין לא סומנו, ירוקים מוכנים ל-commit.

>_ git status
2 add

מסמנים קבצים

מוסיפים לסטייג'ינג — "add ." אומר ״הכל״, אפשר גם לציין קובץ ספציפי.

>_ git add .
3 commit

שומרים snapshot

יוצרים commit עם הודעה תיאורית קצרה — מה השתנה ולמה. זה ישמר בהיסטוריה.

>_ git commit -m "add login page"

מסנכרנים עם GitHub

1 push

שולחים ל-GitHub

שולחים את ה-commits שלנו ל-GitHub — מעדכנים את הענן עם הגרסה העדכנית שלנו.

>_ git push
2 pull

מושכים מ-GitHub

מביאים שינויים חדשים מ-GitHub — מישהו אחר דחף קוד, או שינינו משכשהו אחר.

>_ git pull
3 log

רואים היסטוריה

מציגים רשימה קצרה של ה-commits האחרונים — מי שינה מה ומתי.

>_ git log --oneline
