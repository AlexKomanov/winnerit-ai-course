מה זה Gem?

Gem הוא גרסה מותאמת אישית של Gemini — עם הוראות קבועות, תפקיד מוגדר, וידע ספציפי שנטען מראש.

במקום לכתוב מחדש את ה-prompt בכל שיחה, אנחנו מגדירים פעם אחת ומשתמשים שוב ושוב.

מקור: gemini.google.com/overview/gems

יצירת Gem

1 navigation

פותחים את Gemini

נכנסים לכתובת gemini.google.com מחשבון Google. פותחים את ה-sidebar ומוצאים את קטגוריית Gems.

🌐 gemini.google.com
2 create

New Gem

לוחצים Explore Gems ואז New Gem. נותנים שם ומזינים הוראות — Persona, Task, Context, Format. ניתן גם ללחוץ על ✦ Re-write ו-Gemini יעזור להרחיב.

✦ Use Gemini to re-write instructions
3 preview & save

בדיקה ושמירה

בחלונית הימנית בודקים את ה-Gem עם prompt לדוגמה לפני השמירה. לאחר שמחים — לוחצים Save. ה-Gem יופיע תחת My Gems ב-sidebar.

💾 Save → My Gems

Gem לבודק QA

הגדרת ה-Gem

מה שמגדירים

persona

בודק תוכנה בכיר, ניסיון ב-Agile, מכיר את ה-stack של הפרויקט

task

כתיבת מקרי בדיקה, ניתוח edge cases, זיהוי פערים בין PRD לבין ה-implementation

context

שם הפרויקט, סביבת העבודה, הצוות, מגבלות ידועות

format

טבלת מקרי בדיקה בעברית: מס׳ / תיאור / קלט / פלט צפוי / עדיפות / סטטוס
