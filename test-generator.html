מה זה PRD?

PRD (Product Requirements Document) הוא מסמך שמתאר מה המוצר צריך לעשות — features, user flows, edge cases, וקריטריוני קבלה.

כשה-AI קורא PRD טוב, הוא מבין בדיוק מה לבדוק — ויכול לכתוב את הבדיקות בעצמו.

requirements → tests · automated

הflow האוטומטי

1 prd

PRD

Requirements

מסמך דרישות ברור — user stories, acceptance criteria, edge cases. ככל שהמסמך מדויק יותר, הבדיקות איכותיות יותר.

2 agent

Agent

AI Reader

Claude Code agent קורא את ה-PRD, מזהה תרחישים ויוצר רשימת test cases מסודרת.

3 generate

Generate

Test Code

ה-Agent כותב קבצי Playwright — test files מלאים, עם selectors, assertions ו-edge cases.

4 run

Run

Execute

מריצים את הבדיקות עם Claude Code CLI. ה-Agent מפרש שגיאות ומתקן אוטומטית.

מכינים את הסביבה

1 prd

מכינים PRD

כותבים או מייבאים מסמך PRD. חשוב לכלול: user flows, acceptance criteria, וקריטריוני הצלחה.

→ PRD.md — מסמך דרישות בפורמט Markdown
2 playwright

מתקינים Playwright

מוסיפים Playwright לפרויקט. Claude Code ידאג לקבצי הגדרה בסיסיים.

>_ npx playwright install
3 skill

מגדירים את ה-Skill

מגדירים Skill שמנחה את Claude Code לקרוא PRD וליצור test cases לפי פורמט הפרויקט.

→ .claude/test-generator.md — Skill מותאם

מריצים את הGenerator

1 invoke

מפעילים את ה-Agent

מפעילים את Claude Code עם ה-Skill ומצביעים על קובץ ה-PRD. ה-Agent מתחיל לקרוא ולנתח.

→ /test-generator PRD.md — מפעיל את ה-Skill
2 review

בודקים את התוצאות

ה-Agent מציג את ה-test cases שייצר לפני כתיבתם. אנחנו יכולים לאשר, לדחות, או לבקש שינויים.

→ Claude Code מציג test plan לאישור
3 run

מריצים ומתקנים

אחרי אישור, ה-Agent כותב את הקבצים ומריץ. שגיאות מפורשות ומתוקנות אוטומטית.

>_ npx playwright test
