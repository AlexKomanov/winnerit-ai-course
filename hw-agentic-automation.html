מתחילים עם Linear

1linear account

יוצרים חשבון Linear

נרשמים ב-linear.app ומגדירים workspace חדש לצוות שלנו. נצור פרויקט ראשון בשם המוצר שאנחנו בונים.

2linear mcp

Linear MCP ב-Claude Code

נחבר את Linear ל-Claude Code דרך ה-MCP. כך יוכל Claude Code לנהל טיקטים, לעדכן סטטוסים ולפתוח באגים — הכל ישירות מהצ'אט.

3playwright mcp

Playwright MCP ב-Claude Code

נחבר את Playwright ל-Claude Code דרך ה-MCP. כך יוכל Claude Code להפעיל דפדפן, לנווט בין דפים ולהריץ בדיקות על האפליקציה שלנו.

הפרויקט וה-PRD

1open project

פותחים את הפרויקט

פותחים Claude Code בתוך תיקיית הפרויקט מהשיעור הקודם. Claude Code יזהה את הקוד, יקרא את מבנה הפרויקט ויכין את עצמו לעבודה.

2add prd

מוסיפים את ה-PRD

מעתיקים את קובץ ה-PRD מהשיעור הקודם לתיקיית הפרויקט ושומרים אותו בתור PRD.md. זהו ה"תקציר" שינחה את כל הבדיקות שניצור.

פרויקט QA וטיקטים

1qa project

פרויקט QA חדש ב-Linear

מבקשים מ-Claude Code ליצור פרויקט חדש ב-Linear ייעודי לאוטומציה — נפריד בין טיקטי פיתוח לטיקטי QA.

› Create a new Linear project called "QA Automation" for tracking all test automation tickets
2create tickets

טיקטים מה-PRD

מבקשים מ-Claude Code לקרוא את PRD.md וליצור טיקטי QA לכל פיצ'ר — כל טיקט יכלול שלבי בדיקה ממוספרים וקריטריוני הצלחה ברורים.

› Read PRD.md and create QA automation tickets in the "QA Automation" Linear project. Each ticket must include: what to test, numbered test steps, and expected results

בדיקה וכיוונון

1review

סקירת הטיקטים ב-Linear

נפתח את פרויקט ה-QA ב-Linear ונעבור על כל הטיקטים שנוצרו. נבדוק שלבי בדיקה, קריטריוני הצלחה ותיעדוף.

Linear → QA Automation → All Issues → review each ticket
2correct

עדכון ותיקון

אם טיקט לא מדויק — נכתוב ל-Claude Code מה לשנות. אפשר לתקן כותרת, להוסיף שלבים, לפרט edge cases או לשנות priority.

› Update the QA ticket for [feature]: revise the test steps to include [scenario], add edge cases for empty state, and set priority to High

כותבים ומריצים

1write tests

כותבים בדיקות E2E

מבקשים מ-Claude Code לקחת את הטיקטים מה-Linear QA ולכתוב בדיקות Playwright לכל אחד. הוא יסמן כל טיקט "In Progress" בזמן שהוא כותב.

› Take all open tickets from the Linear "QA Automation" project and write Playwright E2E tests for each one. Mark each ticket "In Progress" when you start working on it
2run

מריצים ורואים

מריצים את כל הבדיקות ועוקבים אחרי התוצאות — מה עבר, מה נכשל, ואיפה יש בעיות באפליקציה.

› npx playwright test --reporter=list

סטטוסים ובאגים

1update status

עדכון סטטוסים

מבקשים מ-Claude Code לעבור על תוצאות הבדיקות ולסמן כל טיקט שעבר בהצלחה כ-"Done" בפרויקט ה-QA ב-Linear.

› Review the test results and mark all passing test tickets as "Done" in the Linear QA Automation project
2open bugs

פותחים באגים ב-Linear

עבור כל בדיקה שנכשלה, מבקשים מ-Claude Code לפתוח טיקט באג ב-Linear עם label "bug" ופרטי הכישלון.

› For each failing Playwright test, create a Bug ticket in Linear with label "bug", include the error message and the expected vs actual behavior
