מה זה MCP?

Model Context Protocol — פרוטוקול פתוח שמאפשר ל-AI לגשת לכלים חיצוניים בצורה מבוקרת ומאובטחת. כל פעולה דורשת אישור מפורש שלכם לפני הביצוע.

Claude Desktop הוא לקוח שתומך ב-MCP — ואנחנו נחבר אותו לשרת Playwright, שנותן לו גישה לדפדפן אמיתי.

MCP server → Claude Desktop → browser → your app

מה Playwright MCP נותן

ניווט בדפדפן

מבקרים בכל עמוד ונתיב באפליקציה — כולל עמודים שדורשים login, ניווט עמוק ו-dynamic routes.

אינטראקציה מלאה

לוחצים על כפתורים, ממלאים טפסים ומשלימים flows מקצה לקצה — בדיוק כמו משתמש אנושי.

תיעוד חזותי

מצלם screenshots בזמן אמת, מנתח ממשק משתמש ומייצר מיפוי מלא של כל state ו-UI pattern.

מחברים ל-Claude Desktop

1 settings

פותחים את ה-Settings

פותחים את Claude Desktop, עוברים ל-Developer ולוחצים על Edit Config.

Claude → Settings → Developer → Edit Config

מוסיפים את ה-Config

מדביקים את ה-JSON הבא לקובץ ושומרים.

claude_desktop_config.json
{ "mcpServers": { "playwright": { "command": "npx", "args": [ "@playwright/mcp@latest" ] } } }

הפרומפט ל-PRD

prd prompt · paste into claude desktop
You are a senior product manager and UX researcher. I will give you a URL. Your job is to: 1. EXPLORE — Use Playwright to systematically visit every page and route. Click every navigation link, button, and CTA. Complete every user flow end-to-end (sign up, login, create, edit, delete, purchase, submit — whatever applies). Test empty states, populated states, and edge cases like removing items or going back mid-flow. Take note of every interaction, state change, and UI pattern you encounter. 2. DOCUMENT — After full exploration, produce a comprehensive PRD as a professional Word (.docx) file with color-coded tables and clear section hierarchy, covering: Executive Summary Product Overview (vision, target audience, business goals) Site Map & Information Architecture (every route discovered) Feature Catalogue (every feature, every page, every interaction) User Flows (step-by-step, every flow you completed) Functional Requirements (with ID, feature name, description, priority: High / Medium / Low) Non-Functional Requirements (performance, accessibility, security, responsiveness) UX & Design Specifications (layout patterns, design tokens, interaction patterns) Identified Gaps & Recommendations (what's missing, what should be built next) Glossary Be thorough. Do not stop exploring until you have visited every reachable page and completed every available user flow. The URL is: [PASTE URL HERE]

שומרים את ה-PRD

1 local

שמירה מקומית

Claude מייצר קובץ .docx ושומר אותו במחשב. עובד מצוין לפעם אחת — אבל בכל שינוי באפליקציה צריך לחזור ולהפיק מחדש.

שמור קובץ .docx בתיקיית הפרויקט
2 recommended

העלאה ל-Google Docs

מעלים את ה-PRD ל-Google Docs ומחברים אותו כ-Knowledge ב-Gem. המסמך נשאר עדכני — כל עדכון ב-PRD מגיע אוטומטית ל-Gem.
וה-test cases תמיד מבוססים על הגרסה הנוכחית.

Google Docs → Share link → Gem → Knowledge → Add URL
